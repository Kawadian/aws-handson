# SQS ハンズオン

キュー投入・受信・再試行を体験し、非同期処理の基本を理解する教材です。

## ゴール

- 標準キューと DLQ を作成する
- メッセージ送信と受信を試す
- 失敗時に再試行や退避が必要になる理由を理解する

## 作るもの

- `handson-task-queue`
- `handson-task-dlq`

## 手順

### 1. キューを作成する

1. SQS コンソールを開く
2. `handson-task-dlq` を先に作成する
3. 続けて `handson-task-queue` を作成する
4. `handson-task-queue` の Redrive policy で DLQ に `handson-task-dlq` を設定する

### 2. メッセージを送る

本文に次のような JSON を入れて送信します。

```json
{
  "taskId": "task-001",
  "action": "created",
  "userId": "user-001"
}
```

### 3. メッセージを受信する

1. **メッセージの送受信** を開く
2. メッセージをポーリングして内容を確認する
3. 受信後に削除しなければ再度見えるようになることを確認する

### 4. 非同期処理を考える

- すぐ終わらない処理をなぜ API 本体から切り離すのか
- ワーカー失敗時に DLQ があると何が助かるのか

## 発展課題

- Lambda トリガーを付けて自動処理する
- 可視性タイムアウトを変えて挙動を比較する

## 片付け

- `handson-task-queue` と `handson-task-dlq` を削除する

## 学べるポイント

- SQS は処理の分離とバースト吸収に向いている
- 「送信成功」と「処理成功」は別であることを体験できる
- DLQ を使うと失敗メッセージを後から調査しやすい
