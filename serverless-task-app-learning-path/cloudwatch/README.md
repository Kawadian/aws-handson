# CloudWatch ハンズオン

Lambda のログ・メトリクス・アラームを確認し、運用の入口を体験する教材です。

## ゴール

- Lambda の実行ログを確認する
- メトリクスをダッシュボードで見る
- エラーアラームを作成して SNS 通知とつなげる

## 前提条件

- [Lambda ハンズオン](../lambda/README.md) を実施済み
- 可能なら [SNS ハンズオン](../sns/README.md) も実施済み

## 手順

### 1. CloudWatch Logs を確認する

1. CloudWatch コンソールを開く
2. **ロググループ** から `/aws/lambda/handson-hello-lambda` を開く
3. 最新のログストリームを開いて実行内容を確認する

### 2. メトリクスを見る

1. **メトリクス > Lambda** を開く
2. `Invocations` と `Errors` を表示する
3. Lambda を数回手動実行してグラフが増えることを確認する

### 3. アラームを作る

1. `Errors` メトリクスから **アラームを作成** を選ぶ
2. 条件は **1 回以上** にする
3. 通知先に SNS トピック `handson-task-events` を設定する
4. Lambda コードを一時的に例外送出する形に変え、アラーム遷移を確認する

### 4. ダッシュボードを作る

1. **ダッシュボード** を作成する
2. `Invocations` と `Errors` の 2 つを並べる
3. 監視画面として見やすいか確認する

## 片付け

- 作成したアラームを削除する
- 作成したダッシュボードを削除する

## 学べるポイント

- CloudWatch は「ログを見る場所」だけでなく監視の土台でもある
- 正常系だけでなく、失敗時にどう気づくかまで考えるのが運用設計
- SNS と組み合わせると通知まで一連で理解できる
