# Cognito ハンズオン

ユーザー登録・確認・ログインを体験し、アプリ認証の基本を理解する教材です。

## ゴール

- Cognito User Pool を作成する
- ユーザー登録とメール確認を試す
- Hosted UI でログインを体験する

## 手順

### 1. User Pool を作成する

1. Cognito コンソールを開く
2. **ユーザープールを作成** を選ぶ
3. サインインオプションは **メールアドレス** にする
4. パスワードポリシーはデフォルトのままでよい
5. アプリケーションクライアント名を `handson-web-client` にする
6. Hosted UI を有効にする

### 2. テストユーザーを登録する

1. Hosted UI を開く
2. **Sign up** からメールアドレスとパスワードを登録する
3. メールに届く確認コードでアカウントを有効化する

### 3. ログインを試す

1. Hosted UI に戻ってログインする
2. ログイン後、発行された ID トークン / Access Token の存在を確認する

### 4. アプリ連携を考える

- API Gateway にトークンを渡すとどう認証されるか
- Lambda 側で `sub` や `email` をどう使うか

## 発展課題

- ユーザー属性に `name` を追加する
- パスワード再設定の流れを確認する

## 片付け

- User Pool とアプリクライアントを削除する

## 学べるポイント

- Cognito は会員登録・ログイン・トークン発行をまとめて提供する
- ID トークンを API Gateway やフロントエンドでどう扱うかが重要
- 認証を個別実装しないことで開発を大きく簡略化できる
