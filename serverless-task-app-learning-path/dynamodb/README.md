# DynamoDB ハンズオン

シンプルなタスクテーブルを作成し、項目追加・取得・Query を体験する教材です。

## ゴール

- DynamoDB テーブルを作成する
- パーティションキーとソートキーの役割を理解する
- アイテムの追加と Query を試す

## 作るもの

- テーブル名: `HandsonTasks`
- パーティションキー: `userId` (String)
- ソートキー: `taskId` (String)

## 手順

### 1. テーブルを作成する

1. DynamoDB コンソールを開く
2. **テーブルの作成** を選ぶ
3. 次の内容で作成する
   - テーブル名: `HandsonTasks`
   - パーティションキー: `userId`
   - ソートキー: `taskId`
4. キャパシティモードは **オンデマンド** のままで作成する

### 2. アイテムを 2 件追加する

| userId | taskId | title | status |
|--------|--------|-------|--------|
| user-001 | task-001 | IAM を学ぶ | todo |
| user-001 | task-002 | Lambda を学ぶ | doing |

### 3. Query を試す

1. **項目を探索** を開く
2. `userId = user-001` で Query を実行する
3. 同じユーザーのタスクだけが返ることを確認する

### 4. データモデルを考える

- `status` をキーに含めない理由は何か
- 別ユーザーのデータをどう分離するか
- TTL を付けるならどの属性を追加するか

## 発展課題

- `dueDate` を追加して期限付きタスクにする
- Lambda から PutItem / Query する構成に広げる

## 片付け

- テーブル `HandsonTasks` を削除する

## 学べるポイント

- DynamoDB はアクセスパターンからキー設計を逆算するのが基本
- Query と Scan の違いを体験しておくと設計しやすい
- 小さなテーブルでも本番アプリのデータ設計の入口になる
